## How to Contribute

1. Fork this repository to your own GitHub account.  
2. Create a new branch for each feature, fix, or improvement:  
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. Make your changes in the branch.
4. Commit your changes using **Conventional Commits**.
5. Push your branch to your forked repository.
6. Open a Pull Request (PR) from your branch to the main branch of this repository.

> Each feature, fix, or improvement should be in a separate branch with its own PR. This allows for easier review and integration.

---

## Pull Requests
- Follow the GitHub Flow as the primary versioning workflow.

- Use descriptive commit messages following **Conventional Commits**.

- Include a brief description of your changes and why they are needed in your PR.

---

## Style Guide
- All contributions should adhere to:

- **Conventional Commits** for commit messages.

- Consistent code formatting and readability.

- Clear documentation for new features, projects, or session notes.

- Professional, respectful, and constructive communication in comments and discussions.

---

## Ways to Contribute
- Bug Reports and Typos
  Report any bugs, errors, or inconsistencies to help maintain a clean and error-free repository.

- Session Notes
  Contribute summaries or notes from cohort preparation sessions to enhance collective learning.

- Project Ideas and Tips
  Share creative project ideas, coding tips, and best practices for fellow learners.

- Anything Valuable
  Any contribution that adds value to the repository or promotes collaborative learning is welcome.
